In the non-editing view (the `<div className="cursor-text" onClick={() => setIsEditing(true)}>` section), after the scene chips block (after the closing `)}` of the `element.scene_links.length > 0 &&` conditional), add:

```tsx
{category === 'character' && !isEditing && (
  <div className="mt-2 flex items-center" onClick={e => e.stopPropagation()}>
    {element.synced_to_characters || syncMutation.isSuccess ? (
      <span className="inline-flex items-center gap-1 text-[10px] text-muted-foreground/60">
        <UserCheck className="h-3 w-3" />
        Synced
      </span>
    ) : (
      <button
        onClick={() => syncMutation.mutate()}
        disabled={syncMutation.isPending}
        className="text-[10px] text-emerald-400/80 hover:text-emerald-400
          disabled:opacity-40 disabled:cursor-not-allowed transition-colors"
      >
        {syncMutation.isPending ? 'Adding...' : '+ Add to Characters'}
      </button>
    )}
  </div>
)}
```

Note on `syncMutation.isSuccess`: This provides instant visual feedback before the `onSettled` re-fetch resolves, covering both `created` and `already_exists` response paths (both return 200 so mutation succeeds). The re-fetched element will have `synced_to_characters: true` set in the backend, locking the Synced state permanently.

The `onClick={e => e.stopPropagation()}` on the wrapper div prevents the click from bubbling to the card's `onClick={() => setIsEditing(true)}` handler.
