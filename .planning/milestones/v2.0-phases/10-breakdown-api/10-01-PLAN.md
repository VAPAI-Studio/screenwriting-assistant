**Imports** (follow existing endpoint import pattern from list_items.py):


**Two helper functions:**

1. `_verify_project_ownership(db, project_id, user_id)` -- query Project by id + owner_id, return project or raise 404. Pattern from list_items.py.

2. `_verify_element_ownership(db, element_id, user_id)` -- query BreakdownElement by id, raise 404 if not found, call `_verify_project_ownership` to verify the element's project belongs to user. Return element.

**Four endpoint functions:**

1. `GET /elements/{project_id}` (API-02):
   - `response_model=List[schemas.BreakdownElementResponse]`
   - Query params: `category: Optional[str] = None`, `include_deleted: bool = False`
   - Call `_verify_project_ownership`, then query `BreakdownElement` filtered by project_id
   - Default: filter `is_deleted == False` (only include deleted if `include_deleted=True`)
   - If `category` provided, add filter on category
   - Order by `sort_order` then `created_at`
   - Return `.all()`

2. `POST /elements/{project_id}` (API-04):
   - `response_model=schemas.BreakdownElementResponse`, `status_code=status.HTTP_201_CREATED`
   - Body: `schemas.BreakdownElementCreate`
   - Call `_verify_project_ownership`
   - BEFORE inserting, check for existing element with same (project_id, category, name) -- INCLUDING soft-deleted
   - If existing and `is_deleted=True`: RESTORE it (set `is_deleted=False`, update description/metadata from body, set `source='user'`, set `user_modified=True`), commit, return existing
   - If existing and NOT deleted: raise 409 Conflict with message "Element '{name}' already exists in category '{category}'"
   - Otherwise: create new `BreakdownElement` with `source='user'`, commit, refresh, return
   - Map body.metadata to ORM attribute `metadata_`

3. `PUT /element/{element_id}` (API-03):
   - `response_model=schemas.BreakdownElementResponse`
   - Body: `schemas.BreakdownElementUpdate`
   - Call `_verify_element_ownership`
   - Use `body.model_dump(exclude_unset=True)` for partial update
   - Map `metadata` key to `metadata_` ORM attribute (if "metadata" in update_data, pop it and assign to "metadata_")
   - Set each field via `setattr(element, field, value)`
   - ALWAYS set `element.user_modified = True` (critical for API-03)
   - Commit, refresh, return element

4. `DELETE /element/{element_id}` (API-05):
   - Return 200 with `{"status": "success", "message": "Element soft-deleted"}`
   - Call `_verify_element_ownership`
   - Set `element.is_deleted = True` (NEVER use `db.delete()`)
   - Commit

Do NOT add scene link, summary, or extraction endpoints -- those are Plan 02.
