# Milestone v8.0: MCP Server

**Status:** ✅ SHIPPED 2026-06-12
**Phases:** 55-61
**Total Plans:** 3 (Phase 55) + inline execution (56-61)

## Overview

Exposed the app's core capabilities as **17 MCP tools** over a remote Streamable HTTP server mounted in-process on the existing FastAPI app at `/mcp`, authenticated by the existing v5.0 `sa_<key>` API-key gateway (static bearer, no OAuth). All tools are owner-scoped; long-running AI tools use a job-id + poll pattern; no delete tools are exposed. Consumers: Claude Code / Claude Desktop (primary), Hermes (secondary). Verified end-to-end live from Claude Code (`whoami` returned the key owner).

## Phases

### Phase 55: MCP Foundation — Mount, Auth, Lifespan & Client Spike
**Requirements:** MCPF-01..05 · **Verification:** passed
Mounted the MCP Streamable HTTP server in-process at `/mcp`; extracted `authenticate_token` out of `get_current_user` (shared by REST + MCP, carrying the `request_count` increment); composed the MCP session-manager lifespan into the app lifespan; exempted `/mcp` from the BaseHTTPMiddleware stack; `ping` + `whoami` tools. ## Milestone Summary

**Key Decisions** (full log in `.planning/v8.0-AUTONOMOUS-DECISIONS.md`):
- Official `mcp` SDK over standalone fastmcp; no FastAPI bump (pinned starlette/sse-starlette).
- Mount in-process; exempt `/mcp` from BaseHTTPMiddleware; compose the MCP sub-app's lifespan.
- Same v5.0 `sa_<key>` keys for MCP; a valid key grants all tools (per-tool scopes → v8.1); owner-scoped.
- In-memory job registry (not a DB table) for long-running tools.
- Package named `mcp_server/` (not `mcp/`) to avoid shadowing the SDK.
- No delete tools (locked).

**Tool surface (17):** ping, whoami, job_status · project_list/get/create, show_list, show_read_bible, episode_list · screenplay_read/write/generate_scene · breakdown_extract/read · shotlist_read, shot_create, shotlist_generate.

**Issues Resolved:** dependency conflict (sse-starlette pulling starlette 1.x), `/mcp` trailing-slash redirect, DNS-rebinding 421, single-use session-manager test isolation (`SKIP_MCP_LIFESPAN` flag + session-factory override), enum-or-str column coercion.

**Technical Debt / Flagged for Review:**
- Legacy `framework` Postgres enum is broken app-wide (uppercase ORM names vs lowercase PG labels) — worked around in MCP, worth a schema fix (D-57-A).
- Dependency pinning (starlette/sse-starlette) to avoid a FastAPI bump — confirm with a clean `docker compose build` (D-55-B2).
- In-memory job registry is per-worker (single uvicorn worker assumed; Redis/DB if scaled).
- DNS-rebinding protection off by default (env-overridable).
- `/mcp` requires a backend restart to activate if the process started before the v8.0 code.
- Hermes static-header support unverified (defers to v8.1 if unsupported — not a blocker).

**Stats:** 7 phases, 14 commits, ~25 new MCP tests across 7 files, all on 2026-06-11/12. Full suite ~399 passed (only the 4 documented pre-existing flakes fail). Live UAT: Claude Code → whoami passed.
