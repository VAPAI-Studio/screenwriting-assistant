### Integration Issues (Non-Blocking)

**1. SELC-04 — ScriptReadView scene mapping heuristic (moderate)**

File: `frontend/src/components/Breakdown/ScriptReadView.tsx:191`

```tsx
const sceneItem = sceneItems?.find(si => si.sort_order === idx) ?? sceneItems?.[idx];
```

Matches screenplay entries (indexed 0, 1, 2...) to scene `ListItem` records by `sort_order`. If `sort_order` values are non-contiguous (gaps from deletion/reordering), shots created from text selection may store incorrect `scene_item_id`. Shot still creates successfully; it may appear under a wrong scene group.

Severity: Moderate data quality issue in edge cases. Does not block core functionality.

**2. SYNC-01 — reorder_list_items missing staleness hook (minor)**

File: `backend/app/api/endpoints/list_items.py:~202`

The `reorder_list_items` endpoint changes scene order but does not call `_mark_shotlist_stale`. The other 3 mutation paths (create, update, delete for scene/character items) all call it. Reordering scenes does not invalidate the shotlist staleness flag.

Severity: Minor correctness gap. Reordering scenes is an uncommon path that wouldn't typically invalidate shot-to-scene associations.

**3. MDIA-05 — Delete media has no frontend UI (intentional deferral)**

Backend DELETE `/api/media/{project_id}/{media_id}` and `api.deleteMedia()` exist, but no component calls them. Explicitly deferred per Phase 23 UI-SPEC.

Severity: Intentional. Not a wiring break.
