# Milestone v6.0: Script Quality

**Status:** ✅ SHIPPED 2026-06-11
**Phases:** 45-49
**Total Plans:** 6

## Overview

Deepen the craft quality of AI-generated screenplays across five dimensions — continuity, format fidelity, character voice, screenwriting craft, and a side-by-side quality compare. Internal tool: the goal is output quality, not new feature surface. No market/export/collab features. v6.0 deepened the *script*; the sibling v7.0 deepened the *breakdown* extracted from it.

## Phases

### Phase 45: Continuity-Aware Generation

**Goal**: Each scene's screenplay is generated with awareness of what was actually written before, so tone, voice, and setup/payoff stay consistent across the scene sequence
**Depends on**: Phase 44 (current generation path baseline)
**Requirements**: CONT-01, CONT-02, CONT-03
**Plans**: 1 plan

Plans:
- [x] 45-01: Thread running synopsis + prev-scene text through `_generate_scripts`, add synopsis-update helper, persist synopsis, and continuity tests

**Details:** When `_generate_scripts` writes a scene, the prompt now includes the verbatim text of the immediately-preceding scene plus a maintained running prose synopsis ("story so far"), re-summarized to a word cap via a separate `chat_completion` call. The synopsis advances only on a successful scene so a failed scene cannot poison continuity. Final synopsis persisted into the existing `screenplay_editor` PhaseData.content JSON (no migration). First/single scene gets no continuity block. VERIFICATION: passed (7/7).

### Phase 46: Format Fidelity (Native vs JSON Mode)

**Goal**: Screenplay output preserves industry-standard formatting, with the generation call shape settled to whichever yields better formatting
**Depends on**: Phase 45 (continuity rework settles the generation call shape)
**Requirements**: FMT-01, FMT-02
**Plans**: 1 plan

Plans:
- [x] 46-01: Migrate scene-writing to native output (json_mode=False), TITLE-line title parsing, strengthened layout prompt; preserve Phase 45 contract/continuity/failure

**Details:** The scene-writing call switched to native output (`json_mode=False`) so JSON string-encoding no longer degrades screenplay formatting; the per-screenplay title is parsed off a `TITLE:` line with a scene-summary fallback (never fails on a missing title). Native path adopted outright (no runtime A/B toggle). Works for both OpenAI and Anthropic via the existing provider abstraction. VERIFICATION: passed (9/9, +1 IN-01 fix).

### Phase 47: Character Voice Injection

**Goal**: Each character speaks in a distinct, consistent voice in generated dialogue because their voice profile reaches the script-writing prompt, not just scene planning
**Depends on**: Phase 46 (generation call shape settled), Phase 45 (continuity context)
**Requirements**: VOICE-01, VOICE-02, VOICE-03
**Plans**: 1 plan

Plans:
- [x] 47-01: Inject character voice profiles into the script-writing prompt (wizards.py guard + `_generate_scripts`) with no-regression tests

**Details:** Per-character voice/diction profiles (from PhaseData story.characters ListItems) are injected into the script-writing prompt in `_generate_scripts`, not only into `_generate_scenes`. The conditional `character_block` collapses to `''` when no characters are present → byte-identical Phase 46 prompt. Under-specified voices are derived and carried forward by the Phase 45 continuity block (no structured voice ledger). VERIFICATION: passed (5/5).

### Phase 48: Screenwriting Craft Guidance

**Goal**: Generated screenplays reflect explicit craft direction so action lines are visual and economical and dialogue carries subtext
**Depends on**: Phase 47 (voice profiles in the prompt), Phase 46 (settled call shape)
**Requirements**: CRAFT-01, CRAFT-02, CRAFT-03
**Plans**: 1 plan

Plans:
- [x] 48-01: Add an unconditional `## Screenwriting Craft` block (subtext, action economy, show-don't-tell, white space) to the `_generate_scripts` prompt + tests

**Details:** `_generate_scripts` carries an unconditional `## Screenwriting Craft` block (subtext / on-the-nose, economical action, show-don't-tell + "no internal or unfilmable description", white-space pacing) added equally to both character paths so Phase 47's byte-identical empty-vs-absent contract holds. Craft anchors chosen not to collide with continuity or voice markers. VERIFICATION: passed — craft-obedience (CRAFT-02/03) judged via the Phase 49 side-by-side compare, UAT confirmed 2026-06-11.

### Phase 49: Side-by-Side Quality Compare

**Goal**: The user can directly compare a scene regenerated with the improved path against its prior output to judge the cumulative quality improvement
**Depends on**: Phase 48, 47, 46, 45 (improved generation path complete)
**Requirements**: EVAL-01
**Plans**: 2 plans

Plans:
- [x] 49-01: Backend — `_generate_one_scene` helper + regenerate-scene (preview) & keep-scene-version (persist) endpoints + tests
- [x] 49-02: Frontend — regenerateScene/keepSceneVersion client + SceneCompareModal + per-scene trigger

**Details:** Extracted `_generate_one_scene` as the single shared per-scene prompt source; both the `_generate_scripts` loop and the new `regenerate_single_scene` delegate to it (no divergent prompt). `POST /api/wizards/regenerate-scene` returns a preview (no DB write); `POST /api/wizards/keep-scene-version` persists only `screenplays[episode_index]` + the matching ScreenplayContent row and marks breakdown/shotlist stale. `episode_index` is the implicit scene key (no migration). Frontend `SceneCompareModal` shows current-vs-regenerated two-pane compare with keep-current (no-op) / keep-new (persist + invalidate). Backend verified end-to-end 2026-06-11 (regenerate-scene preview regenerated a previously-failed scene — title + 12.5K formatted content — no persist, no stale flip); runtime visual UAT confirmed by user 2026-06-11. VERIFICATION: passed (3/3).

## Milestone Summary

**Key Decisions:**

- Continuity threads a running prose synopsis + the immediately-preceding scene's verbatim text into each later prompt; synopsis advances only on success (a failed scene cannot poison continuity)
- Scene-writing switched to native output (json_mode=False) outright (no runtime A/B toggle) — JSON string-encoding was degrading screenplay formatting; title parsed off a TITLE: line with scene-summary fallback
- Character voice injected via a conditional block that collapses to '' when absent → byte-identical empty-vs-absent prompt contract preserved across phases 47/48
- Craft guidance is an unconditional prompt block whose anchors deliberately avoid colliding with continuity/voice markers
- `_generate_one_scene` is the single shared per-scene prompt source — the batch loop and single-scene regenerate both delegate to it, so they can never diverge
- Single-scene regenerate is preview-only; keep persists + marks stale; episode_index is the scene key (no migration)

**Issues Resolved:**

- JSON-wrapping degradation of screenplay formatting (fixed by native output, Phase 46)
- Prompt divergence risk between batch generation and single-scene regenerate (fixed by the shared `_generate_one_scene` helper, Phase 49)
- ROADMAP/STATE progress drift at close: phases 45/47/48/49 were complete in artifacts but mis-marked in the progress table (reconciled 2026-06-11)

**Issues Deferred / Technical Debt Incurred:**

- ScreenplayContent accumulates duplicate rows per episode_index (batch-generate appends, never deletes — pre-dates v6.0). keep-scene-version orders newest-first (WR-01 fix), but breakdown/shotlist still read all rows unordered. A robust fix (insertion-order column or making ScreenplayContent a pure projection) needs a schema change, deferred per D-49-03.
- Test-suite isolation: `test_yolo_integration.py` + `test_session_isolation.py` are order-sensitive (pre-existing, not a v6.0 regression). See v6.0-PREEXISTING-TEST-CONCERN.md.
- Frontend `npm run lint` references a non-existent ESLint config (none has ever existed in the repo); `tsc`/`npm run build` is the binding type gate.

**Stats:** 5 phases, 6 plans, 50 files changed, ~7,500 insertions, all on 2026-06-06 (UAT + close 2026-06-11). Audit: 11/12 requirements satisfied at audit time; EVAL-01 confirmed by user UAT 2026-06-11 → 12/12.
