## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline cell padding |
| sm | 8px | Table cell internal padding, compact gaps |
| md | 16px | Default element spacing, panel padding |
| lg | 24px | Scene group header padding, section breaks |
| xl | 32px | Empty state vertical gaps |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Minimum row height 40px for comfortable click targets on shot rows. Scene header height 36px.

Source: Existing pattern — BreakdownLayout uses `px-3 py-2` (12px/8px), BreakdownPanel uses `p-4` (16px), ElementCard uses `p-4` (16px). This phase aligns with those.

- Empty state heading uses Body role at semibold (14px 600) — adequate prominence in a sparse, centered layout.

## Component Inventory

Components to build for this phase, with their visual contracts:

### 1. ShotlistPanel (replaces center placeholder in BreakdownLayout)

- Occupies the `flex-1` center area of BreakdownLayout
- Contains: panel header (already exists), scrollable shot table area
- Background: `bg-background` (inherits from breakdown-mode)
- Overflow: `overflow-auto` on the content area below the header

### 2. SceneGroup

- Groups shots by `scene_item_id`
- Scene header bar: `bg-card/60`, `px-4 py-2`, border-bottom `border-border`
- Scene title: Heading role (13px semibold), `text-foreground`
- Shot count badge: Label role (12px), `text-muted-foreground`, right-aligned
- Collapsible: No (all scenes expanded by default; deferred to v3.1)
- Ungrouped shots (null scene_item_id) appear under "Unassigned Shots" header

### 3. ShotRow

- Rendered as a horizontal row within each SceneGroup
- Minimum height: 40px
- Bottom border: `border-border/50`
- Hover state: `bg-card/30`
- Layout: CSS grid with defined column widths (see Column Layout below)
- Cells:
  - Shot number: `text-xs font-semibold text-primary` in a `w-10` fixed column, center-aligned
  - Field cells: `text-sm text-foreground`, left-aligned, single-line with `truncate`, `px-2 py-2`
  - Action cell (right): reorder handle + delete button, `w-20`, right-aligned, visible on row hover only
- Click-to-edit: Clicking a field cell enters inline edit mode for that cell

### 4. InlineEditCell

- Replaces the static text in a ShotRow cell when clicked
- Input element: `bg-surface border border-border rounded px-2 py-1 text-sm text-foreground`
- Focus ring: `focus:ring-1 focus:ring-primary/50 focus:outline-none`
- Save: on Enter key or on blur (after 150ms debounce, matching ElementCard pattern)
- Cancel: on Escape key, restores previous value
- Optimistic update: immediately reflect change in UI, revert on API error (matching ElementCard mutation pattern)

### 5. AddShotButton

- Position: bottom of each SceneGroup, after the last shot row
- Variant: `ghost` button, full-width, `text-muted-foreground hover:text-foreground hover:bg-muted`
- Label: "+ Add Shot" (Label role, 12px semibold)
- Height: 36px
- When no shots exist in a group, this button is part of the empty state instead

### 6. ReorderHandle

- Position: leftmost area of the action cell in each ShotRow
- Icon: `GripVertical` from lucide-react, `h-4 w-4`
- Default state: `text-muted-foreground/30`
- Hover state: `text-muted-foreground`, `cursor-grab`
- Dragging state: `text-primary`, `cursor-grabbing`
- Implementation: Button-based up/down arrows for v3.0 (drag-and-drop deferred to SMGT-01 in v3.1)
  - Up arrow: `ChevronUp` icon, disabled if first in group, `aria-label="Move shot up"`
  - Down arrow: `ChevronDown` icon, disabled if last in group, `aria-label="Move shot down"`
  - Size: `h-6 w-6` touch target, `h-3.5 w-3.5` icon
  - Grip icon (visual indicator): `aria-label="Reorder shot"` (decorative in v3.0 since up/down buttons handle reorder, but label provided for screen reader context)

### 7. DeleteShotButton

- Position: rightmost area of the action cell in each ShotRow
- Default: `Trash2` icon from lucide-react, `h-3.5 w-3.5`, `text-muted-foreground/40`, `aria-label="Delete shot"`
- Hover: `text-red-400`
- Confirmation: Matches ElementCard pattern — first click shows "Delete?" text button in red, auto-dismisses after 3 seconds, second click confirms deletion
- Disabled state during pending mutation: `opacity-30`

### 8. EmptyState (panel-level, when project has zero shots)

- Centered in the panel content area using `flex flex-col items-center justify-center h-full`
- Icon: `List` from lucide-react, `h-10 w-10 text-muted-foreground/30`
- Heading: "No shots yet" -- Body role (14px), semibold (weight 600), `text-foreground`
- Body: "Create your first shot to start building your shotlist." -- Body role (14px), regular (weight 400), `text-muted-foreground`, `max-w-xs text-center`
- CTA button: "Add First Shot" -- primary variant Button, `mt-4`
- Animation: `animate-fade-in` (already defined in tailwind config)

## Column Layout

The shotlist table uses a CSS grid for consistent column alignment.

| Column | Field Key | Width | Alignment | Truncate |
|--------|-----------|-------|-----------|----------|
| # | shot_number | 48px fixed | center | no |
| Size | fields.shot_size | 1fr (min 80px) | left | yes |
| Angle | fields.camera_angle | 1fr (min 80px) | left | yes |
| Movement | fields.camera_movement | 1fr (min 80px) | left | yes |
| Description | fields.description | 2fr (min 120px) | left | yes |
| Action | fields.action | 2fr (min 120px) | left | yes |
| Actions | (controls) | 80px fixed | right | no |

Grid template: `grid-template-columns: 48px repeat(2, minmax(80px, 1fr)) minmax(80px, 1fr) repeat(2, minmax(120px, 2fr)) 80px`

Notes:
- Horizontal scroll enabled when viewport is narrow (`overflow-x-auto` on the table container)
- Column headers: Label role, `text-muted-foreground`, uppercase, `tracking-wider`
- Column header row: sticky top, `bg-background`, `z-10`, border-bottom `border-border`
- Only 5 key fields shown in table; remaining fields (dialogue, sound, characters, environment, props, equipment, lens, notes) are accessible via a row expansion or detail view in a future phase

## Interaction States

### Loading State
- Skeleton rows: 5 rows of `h-10 rounded bg-muted/20 animate-pulse` (matching ElementList pattern)
- Scene header skeletons: `h-9 w-48 rounded bg-muted/20 animate-pulse`

### Error State
- Inline error banner at top of panel content area
- Style: `bg-destructive/10 border border-destructive/20 rounded-lg px-4 py-3`
- Icon: `AlertCircle` from lucide-react, `h-4 w-4 text-destructive`
- Text: Error copy (see Copywriting Contract)
- Retry button: `text-sm text-primary hover:underline`

### Saving State (inline edit)
- Brief `opacity-60` on the cell being saved
- On success: restore full opacity
- On error: flash `border-destructive` on the cell, revert value

### Reorder State
- During reorder API call: both up/down buttons show `opacity-50 pointer-events-none`
- On success: list re-renders with new order
- On error: revert to previous order (optimistic rollback)

## Data Flow

### React Query Keys

Add to `QUERY_KEYS` in `constants.ts`:
```
SHOTS: (projectId: string) => ['shots', projectId] as const
```

### API Integration

| Operation | Method | Endpoint | Optimistic |
|-----------|--------|----------|------------|
| List shots | GET | `/api/shots/{project_id}` | No (initial load) |
| Create shot | POST | `/api/shots/{project_id}` | Yes |
| Update shot | PUT | `/api/shots/{project_id}/{shot_id}` | Yes |
| Delete shot | DELETE | `/api/shots/{project_id}/{shot_id}` | Yes |
| Reorder shots | POST | `/api/shots/{project_id}/reorder` | Yes |

Optimistic update pattern: Follow the exact pattern in `ElementCard.tsx` — `onMutate` cancels queries, snapshots previous data, applies optimistic change; `onError` reverts; `onSettled` invalidates.

### Scene Grouping (Frontend)

Shots arrive as a flat array sorted by `scene_item_id, sort_order`. Frontend groups them by `scene_item_id`:
- Group key: `scene_item_id` (UUID string) or `"unassigned"` for null
- Group display order: scenes appear in the order of their first shot's position in the API response
- Unassigned group appears last

## TypeScript Types

Add to `types/index.ts`:

```typescript
export interface Shot {
  id: string;
  project_id: string;
  scene_item_id: string | null;
  shot_number: number;
  script_text: string;
  script_range: Record<string, unknown>;
  fields: ShotFields;
  sort_order: number;
  source: 'user' | 'ai';
  created_at: string;
  updated_at: string | null;

  camera_angle?: string;
  camera_movement?: string;
  lens?: string;
  description?: string;
  action?: string;
  dialogue?: string;
  sound?: string;
  characters?: string;
  environment?: string;
  props?: string;
  equipment?: string;
  notes?: string;
}

export interface ShotCreate {

  script_text?: string;
  fields?: Partial<ShotFields>;
  sort_order?: number;
  source?: 'user' | 'ai';
}

export interface ShotUpdate {

  shot_number?: number;
  script_text?: string;
  fields?: Partial<ShotFields>;

}
```
