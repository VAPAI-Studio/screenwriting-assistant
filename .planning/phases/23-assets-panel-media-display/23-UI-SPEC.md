# Phase 23 — UI Design Contract

> Visual and interaction contract for the Assets Panel & Media Display phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Radix + Tailwind system) |
| Preset | not applicable |
| Component library | @radix-ui (Tabs, Slot already installed) |
| Icon library | lucide-react 0.314 |
| Font | DM Sans (body), Playfair Display (display), Courier Prime (screenplay) |

**Note:** This phase operates entirely within `.breakdown-mode` CSS scope, which overrides `:root` HSL variables to the cool steel-blue palette. All color references below use the breakdown-mode variable values.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps within audio player controls, inline badge padding |
| sm | 8px | Gap between thumbnail images, gap between toggle buttons |
| compact | 12px | Element card collapsed padding-y, audio player bar padding-x, upload zone margin-top. Tailwind `space-3`. Justified: 12px provides a tighter vertical rhythm for dense card-based layouts where 8px is too cramped and 16px wastes vertical space in a scrollable panel. |
| md | 16px | Element card internal padding, category section padding |
| lg | 24px | Gap between category groups, panel content padding-x |
| xl | 32px | Gap between major panel sections |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target minimum 44px height for audio play/pause/stop buttons on mobile (accessibility).

## Component Inventory

### 1. Left Panel Toggle (ASST-01)

**Location:** Inside `BreakdownLayout.tsx`, replaces static `ScriptReadView` rendering.

**Toggle control spec:**
- Two adjacent buttons: "Script" and "Assets"
- Placed inside the `BreakdownPanel` content area, directly below the existing panel header
- Active button: `text-primary border-b-2 border-primary` (steel blue text + bottom border)
- Inactive button: `text-muted-foreground border-b-2 border-transparent`
- Button height: 36px (`h-9`), padding: `px-4`
- Font: `text-xs font-semibold uppercase tracking-wider` (matches existing panel header style)
- Gap between buttons: 4px (`gap-1`)
- Container has bottom border: `border-b border-border`

**State preservation (ASST-05):**
- Both `ScriptReadView` and `AssetsPanel` are always mounted in the DOM
- Inactive view uses `style={{ display: 'none' }}` to hide without unmounting
- This preserves scroll position, expanded elements, and React Query subscriptions

### 2. AssetsPanel (ASST-02)

**Layout:**
- Vertical stack of category groups
- Outer padding: 0 (scrollable area fills panel content)
- Each category group has a sticky header

**Category group header:**
- Background: `bg-card/30` (subtle, matches existing CategoryTabs pattern)
- Text: `text-xs font-semibold uppercase tracking-wider text-muted-foreground`
- Element count badge: `text-[10px] bg-muted/60 px-1.5 py-0.5 rounded-full tabular-nums` (matches CategoryTabs badge)
- Padding: `px-4 py-2` (8px vertical padding, on-grid)
- Position: `sticky top-0 z-10` within the scrollable container
- Border: `border-b border-border`

**Empty category:** Hidden entirely (do not show categories with 0 elements).

**Category order:** Characters, Locations, Props, Wardrobe, Vehicles (matches `BREAKDOWN_CATEGORIES` constant).

### 3. AssetElementCard (ASST-03)

**Layout:**
- Collapsible card: click header to expand/collapse
- Collapsed state: shows element name + media count badge + chevron
- Expanded state: shows name, description, media grid, upload zone

**Collapsed state:**
- Container: `rounded-lg border border-border/50 hover:border-border bg-card/40 hover:bg-card/60 transition-all cursor-pointer`
- Padding: `px-4 py-3` (12px vertical — uses `compact` spacing token for dense card rhythm)
- Name: `text-sm font-semibold text-foreground truncate`
- Chevron: `lucide-react ChevronRight` (collapsed) / `ChevronDown` (expanded), size `h-4 w-4`, color `text-muted-foreground`
- Media count badge: `text-[10px] bg-muted/60 px-1.5 py-0.5 rounded-full tabular-nums text-muted-foreground` showing total media count (e.g., "3")
- Badge only shown when count > 0

**Expanded state:**
- Container padding: `px-4 py-4`
- Description: `text-xs text-muted-foreground line-clamp-2 mb-3` (12px bottom margin — uses `compact` spacing token, matches ElementCard)
- Media section and upload zone appear below description
- Animate expand: `animate-accordion-down` (existing keyframe)
- Animate collapse: `animate-accordion-up` (existing keyframe)

**Media data fetch:** Only triggers when `isExpanded === true` via React Query `enabled` flag.

### 4. MediaThumbnail (MDIA-03)

**Layout:**
- Grid of thumbnails: `grid grid-cols-3 gap-2` within the expanded element card
- Each thumbnail: 80px x 80px square (`w-20 h-20`)
- Border radius: `rounded-md` (8px)
- Object fit: `object-cover`
- Border: `border border-border/50`
- Image source: `thumbnail_path` from API response (WebP, server-generated)
- Fallback for missing thumbnail: `bg-muted flex items-center justify-center` with `Image` icon from lucide-react at `h-6 w-6 text-muted-foreground/40`
- Hover: `opacity-80 transition-opacity` (subtle dim)
- Click: Opens full-size image (`file_path`) in a new browser tab (`window.open`)

**Loading state:** Skeleton shimmer using existing `animate-shimmer` while image loads.

### 5. AudioPlayer (MDIA-04)

**Layout:**
- Horizontal bar: `flex items-center gap-2 rounded-lg bg-card/60 border border-border/50 px-3 py-2` (12px horizontal padding — uses `compact` spacing token; 8px vertical padding)
- Full width within the element card
- Stack vertically below thumbnail grid with `mt-2` gap

**Controls:**
- Play button: `lucide-react Play` icon, `h-4 w-4`, wrapped in a 32px (`h-8 w-8`) touch target button, `aria-label="Play audio"`
- Pause button: `lucide-react Pause` icon (replaces Play when playing), same sizing, `aria-label="Pause audio"`
- Stop button: `lucide-react Square` icon, `h-3.5 w-3.5`, wrapped in `h-8 w-8` button, `aria-label="Stop audio"`
- Button style: `text-muted-foreground hover:text-foreground transition-colors rounded-md hover:bg-muted`
- Active play state: Play/Pause icon uses `text-primary` (steel blue)

**Filename display:**
- `text-xs text-muted-foreground truncate flex-1` (fills remaining space)
- Shows `original_filename` from API

**Audio overlap prevention:**
- Parent `AssetsPanel` maintains a `currentlyPlayingId` ref
- Starting playback on one player pauses any currently playing player via a callback prop `onPlaybackStart(mediaId)`

### 6. MediaUploadZone (ASST-04)

**Layout:**
- Positioned at bottom of expanded AssetElementCard, below media items
- Margin top: `mt-3` (12px — uses `compact` spacing token)
- Container: `border-2 border-dashed border-border rounded-lg p-4 text-center transition-colors`
- Min height: 80px (`min-h-[80px]`)

**Default state:**
- Icon: `lucide-react Upload`, `h-6 w-6 text-muted-foreground/40` centered
- Text: `text-xs text-muted-foreground` — "Drop files here or click to upload"
- Subtext: `text-[10px] text-muted-foreground/50` — "Images (JPEG, PNG, WebP) and audio (MP3, WAV, M4A) up to 20MB"

**Drag-over state:**
- Border: `border-primary` (steel blue solid, replaces dashed)
- Background: `bg-primary/5`
- Icon color: `text-primary`
- Text: "Drop to upload"

**Uploading state:**
- Text replaced with filename + spinner (`animate-spin-slow` on `Loader2` icon from lucide-react)
- Border: `border-primary/50`

**Error state:**
- Text: `text-destructive text-xs` showing the error message from the backend
- Resets to default state after 5 seconds

**File picker:**
- Hidden `<input type="file" accept="image/jpeg,image/png,image/webp,audio/mpeg,audio/wav,audio/x-m4a" multiple>`
- Clicking the zone triggers the file picker via ref
- `multiple` attribute allows batch upload

**Upload behavior:**
- FormData POST to `/api/media/{project_id}` with `element_id` field
- No `Content-Type` header (browser sets multipart boundary automatically)
- On success: invalidate `ELEMENT_MEDIA` query key to refresh media list
- On error: display backend error message in the upload zone

## Interaction Contracts

### Toggle Behavior (ASST-01, ASST-05)
1. Default view on first visit: "Script" (matches current behavior)
2. Clicking "Assets" immediately shows the assets panel; ScriptReadView is hidden but NOT unmounted
3. Clicking "Script" immediately shows the script view; AssetsPanel is hidden but NOT unmounted
4. Scroll position in both panels is independently preserved across toggles
5. Expanded/collapsed state of AssetElementCards is preserved across toggles

### Element Expand/Collapse (ASST-03)
1. All elements start collapsed
2. Clicking an element header toggles its expanded state
3. Expanding triggers a media data fetch (if not already cached)
4. Multiple elements can be expanded simultaneously
5. Expand/collapse uses accordion animation (200ms ease-out)

### Drag-and-Drop Upload (ASST-04)
1. Dragging a file over the upload zone changes its visual state (drag-over style)
2. Dragging away from the zone reverts to default state
3. Dropping a valid file initiates upload immediately
4. Dropping an invalid file type shows the backend's validation error
5. The upload zone exists within each expanded element card, scoping the upload to that element
6. File picker button is an alternative to drag-and-drop

### Audio Playback (MDIA-04)
1. Clicking Play starts audio playback from current position (or beginning if stopped)
2. Clicking Pause halts playback at current position
3. Clicking Stop halts playback and resets to beginning
4. Starting playback on one audio file automatically pauses any other currently playing audio
5. When audio ends naturally, the button reverts from Pause to Play

### Media Thumbnail Click (MDIA-03)
1. Clicking a thumbnail opens the full-resolution image in a new browser tab
2. No lightbox or modal -- simple `window.open(file_path, '_blank')`

## Vite Proxy Addition

The `/media` path must be added to the Vite dev server proxy to serve uploaded media files in development mode:

```typescript
// vite.config.ts — add /media proxy entry
proxy: {
  '/api': { target: '...', changeOrigin: true },
  '/media': { target: '...', changeOrigin: true },
}
```

This is a functional requirement, not a visual one, but it directly affects whether thumbnails and audio render correctly.
