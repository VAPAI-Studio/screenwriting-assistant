# Library: official `mcp` SDK (locked decision). NOT standalone fastmcp.
from mcp.server.fastmcp import FastMCP, Context
from mcp.server.auth.provider import TokenVerifier, AccessToken
from mcp.server.auth.settings import AuthSettings

# FastMCP construction (verified signature):
#   FastMCP(name, *, token_verifier=..., auth=AuthSettings(...), streamable_http_path="/", json_response=True, stateless_http=False)
# CRITICAL: when token_verifier is set you MUST also pass auth=AuthSettings(...).
#   AuthSettings REQUIRED fields: issuer_url, resource_server_url (use the local base, e.g. "http://localhost:8001").
#   These are metadata only; static-bearer verification is entirely inside TokenVerifier — no OAuth server is created.
# CRITICAL mount-path: streamable_http_app() defaults streamable_http_path="/mcp". Since we app.mount("/mcp", ...),
#   set streamable_http_path="/" on the FastMCP instance so the effective path is /mcp (NOT /mcp/mcp).
# Recommend json_response=True (simpler request/response for whoami/ping; per STACK.md).

# TokenVerifier protocol (verified):
#   class ApiKeyTokenVerifier(TokenVerifier):
#       async def verify_token(self, token: str) -> AccessToken | None
#   Return None -> SDK responds 401. Return AccessToken(token=..., client_id=..., scopes=[...]) to allow.
#   AccessToken fields: token, client_id, scopes, expires_at, resource.

# Inbound header access inside a tool (verified, official SDK):
#   A tool may take a `ctx: Context` param. Inside: req = ctx.request_context.request  (a Starlette Request)
#   then req.headers.get("authorization"). RequestContext exposes `.request`.
#   (No fastmcp get_http_headers() — that is the other library. Use ctx.request_context.request.)

# ASGI app + lifespan (verified):
#   mcp = FastMCP(...); mcp_app = mcp.streamable_http_app()  # returns a Starlette app
#   mcp.session_manager.run() is the async context manager that MUST be entered in the parent lifespan.

# Shared auth core from Plan 01 (backend/app/api/dependencies.py):
#   def authenticate_token(token: str, db: Session) -> schemas.User   # validates + increments request_count for sa_ keys
#   raises AuthException on invalid/expired/missing.

# DB (backend/app/db.py): SessionLocal (sync sessionmaker), init_db().
# main.py current: init_db runs in @app.on_event("startup"); middleware added :54-58; app.mount("/media",...) at :116.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → /mcp transport | Untrusted JSON-RPC + bearer crosses the mounted ASGI boundary; TokenVerifier is the sole gate |
| MCP tool → DB | Tool resolves owner from header; all reads must be scoped to that user (whoami only returns self this phase) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-04 | Spoofing | ApiKeyTokenVerifier.verify_token | mitigate | SDK rejects (401) when verify_token returns None; sa_ validated by sha256 lookup of active, unexpired key; mock-token only in development ENVIRONMENT |
| T-55-05 | Repudiation / usage-blindness | request_count on MCP path | mitigate | require_user calls authenticate_token which performs the atomic increment; integration test asserts request_count rises via the /mcp path (Pitfall 2) |
| T-55-06 | Denial of Service | BaseHTTPMiddleware buffering /mcp streams | mitigate | /mcp short-circuited out of all five BaseHTTPMiddleware dispatch methods before any buffering (Pitfall 3) |
| T-55-07 | Elevation of Privilege | mounted sub-app bypassing Depends auth | mitigate | Auth enforced inside the MCP transport via TokenVerifier + per-tool require_user, not relying on FastAPI Depends (Pitfall 2) |
| T-55-08 | Tampering | stale per-session request context | mitigate | require_user reads ctx.request_context.request.headers fresh per call; no cached user across calls (Pitfall 4 / locked decision) |
</threat_model>
