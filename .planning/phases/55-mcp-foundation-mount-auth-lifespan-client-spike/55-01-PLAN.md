<objective>
Extract the auth logic currently inlined in `get_current_user` into a reusable, `Depends`-free core `authenticate_token(token, db) -> schemas.User` so both REST and the (future) MCP surface share ONE source of truth — including the atomic `request_count` / `last_used_at` increment for `sa_` keys. Mounted ASGI sub-apps do not run FastAPI's `Depends` chain, so MCP must call this core directly. The refactor MUST be behavior-preserving for REST (Pitfall 2: auth/`request_count` bypass; quality gate: REST keeps working byte-identically).

Output: refactored `dependencies.py`, new pytest file, updated `requirements.txt`.
</objective>

Existing exports in backend/app/api/dependencies.py:
- security = HTTPBearer()
- async def get_current_user(credentials=Depends(security), db=Depends(get_db)) -> schemas.User

The sa_ branch currently does, inline:
1. key_hash = hashlib.sha256(token.encode()).hexdigest()
2. query ApiKey by key_hash + is_active == True; if none -> 401 "Invalid API key"
3. if expires_at (tz-stripped) < utcnow() -> 401 "API key expired"
4. atomic: UPDATE api_keys SET request_count = request_count + 1, last_used_at = :now WHERE id = :id ; db.commit()
5. resolve User by api_key.user_id; if none -> 401 "User not found"
6. return schemas.User(id, email, display_name, created_at)

Mock branch: if settings.ENVIRONMENT == "development" and token == "mock-token" -> mock_auth_service.get_current_user()
JWT branch: auth_service.verify_token(token) -> user_id; resolve User; build schemas.User.
