<decisions>
## Implementation Decisions

### Mode Identity
- Third mode is called "Storyboard"
- Color identity: deep purple/violet (distinct from green/screenwriting and steel-blue/breakdown)
- Mode toggle has three options: Screenwriting / Breakdown / Storyboard
- Mode stored in app state same way as existing two-mode toggle

### StoryboardFrame Model
- Links frames to shots via `shot_id` (FK to Shot, ON DELETE CASCADE)
- Fields: `shot_id`, `file_path`, `thumbnail_path`, `file_type` (image/video), `is_selected` (bool), `generation_source` (user/ai), `generation_style` (photorealistic/cinematic/animated/null)
- Multiple frames per shot allowed; exactly one can be `is_selected=true` at a time
- When `is_selected` is set on a frame, others for same shot are set to false
- `generation_style` stored per-frame so we know what style was used when AI generated it

### Project-Level Style Setting
- `storyboard_style` field added to Project model: enum (photorealistic/cinematic/animated), nullable/default null
- Exposed in project settings or storyboard page header (details deferred to Phase 30)
- Used as default prompt style for Phase 31 AI generation

### CRUD API Design
- Base route: `/api/storyboard/{project_id}`
- List frames by shot: `GET /api/storyboard/{project_id}/shots/{shot_id}/frames`
- Create frame: `POST /api/storyboard/{project_id}/shots/{shot_id}/frames` (multipart upload)
- Update is_selected: `PATCH /api/storyboard/{project_id}/frames/{frame_id}` (set is_selected=true, auto-deselects others)
- Delete frame: `DELETE /api/storyboard/{project_id}/frames/{frame_id}` (also deletes file from disk)
- Reuse existing upload infrastructure from Phase 22 (media upload backend)

### Storyboard Page Shell
- Route: same project page, third mode shows StoryboardView component
- Purple identity applied via accent color on mode tab and any chrome elements
- Empty state when no shots exist or no frames: minimal placeholder, no grid yet (Phase 30)
- Page renders and is navigable; content built in Phase 30

### Migration Strategy
- Delta migration (same pattern as Phase 26 shot model changes)
- New table `storyboard_frames`
- New column `storyboard_style` on `projects` table

### Claude's Discretion
- Exact purple CSS color values (choose a distinctive deep purple, e.g., violet-600/purple-700 range)
- Thumbnail generation strategy (can skip thumbnails in Phase 29, just copy file_path to thumbnail_path)
- Exact error handling and validation details
- File storage location for storyboard frames (use same `uploads/` directory as media)

</decisions>
