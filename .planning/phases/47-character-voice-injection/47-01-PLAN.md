<objective>
Make each character speak in a distinct, consistent voice in generated dialogue by routing their character profiles into the script-writing prompt in `_generate_scripts` — not only into `_generate_scenes`. Today `wizards.py` injects `config["_characters"]` for `scene_wizard` only, and `_generate_scripts` never reads `_characters`, so script-writing prompts have no per-character voice guidance.

This plan:
1. Broadens the `wizards.py` injection guard so `script_writer_wizard` also gets `config["_characters"]` (D-47-01).
2. Has `_generate_scripts` read `_characters` and inject a character-voice section plus an explicit distinct/consistent-voice instruction into every per-scene prompt (D-47-02, D-47-03), reusing the existing empty-list-safe `_build_character_section` so an absent/empty list yields byte-identical Phase 46 behavior (D-47-04).
3. Adds tests proving the names and voice instruction reach `_generate_scripts`, and that the no-character path is unchanged.

Purpose: Deliver VOICE-01 (profiles reach the script prompt), VOICE-02 (voice carried/derived across scenes via the Phase 45 continuity block), and VOICE-03 (characters distinguishable) with a surgical, backend-only change that preserves the Phase 45 continuity contract and the Phase 46 native-output contract exactly.

Output: Modified `wizards.py` + `template_ai_service.py`, a new test module, and a SUMMARY.
</objective>
