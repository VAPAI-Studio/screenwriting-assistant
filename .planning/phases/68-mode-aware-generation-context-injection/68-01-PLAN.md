<objective>
Branch episode-generation context on the show's `continuity_mode`. In `connected` mode, append a new `## Prior Episodes` block built from the `episode_summary` of strictly-prior episodes (ordered by `episode_number` ascending, most-recent-8 cap, stale summaries tagged, empty summaries skipped). In `anthology` and `standalone` modes, fall through to today's bible-only behavior. `show_id IS NULL` films stay unchanged.

Purpose: SCONT-02/03/04 — connected episodes get cross-episode continuity context; anthology/standalone get bible only; feature films are untouched. This is the single behavioral seam of v10.0's continuity feature; Phase 69 will later auto-generate and lazy-regenerate the summaries this phase reads.

Output: An extended `build_bible_context` (the one provider-agnostic helper every generation path funnels through) plus a `TestContinuityModeInjection` test class. Zero downstream changes — `_build_project_context` already prepends the returned string verbatim.
</objective>
