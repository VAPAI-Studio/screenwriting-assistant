   ```python
   class ImagenService:
       """Google Vertex AI Imagen integration for storyboard frame generation."""

       def __init__(self, project_id: str, region: str, model_name: str):
           # Store config; lazy-init the model client

       @staticmethod
       def build_prompt(shot_fields: dict, storyboard_style: str | None) -> str:
           """
           Build an image generation prompt from shot fields and style.

           Prompt structure:
           - Style prefix: map storyboard_style to a descriptive style phrase:
             "photorealistic" -> "Photorealistic cinematic film still."
             "cinematic" -> "Cinematic dramatic film still with high contrast lighting."
             "animated" -> "Animation style frame, clean lines, vivid colors."
             None/other -> "Cinematic film still."
           - Shot composition: "Camera: {shot_size}, {camera_angle}."
             (only include fields that are non-empty)
           - Description: "{description}" (from shot_fields.get("description", ""))
           - Action: "Action: {action}" (from shot_fields.get("action", ""))
           - Suffix: "Professional production storyboard frame. No text overlays."

           Return the assembled prompt string. Skip empty fields gracefully.
           If description and action are both empty, return a minimal
           "Cinematic film still, professional storyboard frame." with the style prefix.
           """

       def generate_image(self, prompt: str) -> bytes:
           """
           Call Vertex AI Imagen to generate a single image from the prompt.

           Steps:
           1. Import vertexai and vertexai.preview.vision_models (or google.cloud.aiplatform)
           2. Initialize: vertexai.init(project=self.project_id, location=self.region)
           3. Load model: ImageGenerationModel.from_pretrained(self.model_name)
           4. Generate: response = model.generate_images(prompt=prompt, number_of_images=1)
           5. Extract bytes: response.images[0]._image_bytes (or response[0]._image_bytes)
           6. Return PNG bytes.

           On error (ImportError, API error), raise a descriptive RuntimeError
           with the original exception chained.
           """
   ```

   Key implementation details:
   - Use `vertexai` package (part of `google-cloud-aiplatform`).
   - Import pattern: `import vertexai` and `from vertexai.preview.vision_models import ImageGenerationModel`.
   - The `vertexai.init()` call should happen in `generate_image`, not at module level (avoid import-time side effects).
   - Wrap the entire generation in try/except, catching `Exception` and re-raising as `RuntimeError(f"Imagen generation failed: {e}")`.
   - Do NOT add `google-cloud-aiplatform` to requirements.txt in this task -- the executor should verify it's available or note it for user setup. The package install is a user setup step.

   Implementation steps inside the endpoint:
   a. Call `_verify_project_ownership(db, project_id, current_user.id)` -> get project object.
   b. Call `_verify_shot_in_project(db, shot_id, project_id)` -> get shot object.
   c. Read `shot.fields` (JSON dict) and `project.storyboard_style`.
   d. Instantiate `ImagenService(settings.GOOGLE_CLOUD_PROJECT, settings.IMAGEN_REGION, settings.IMAGEN_MODEL)`.
   e. Build prompt: `prompt = ImagenService.build_prompt(shot.fields or {}, project.storyboard_style)`.
   f. Call `image_bytes = service.generate_image(prompt)`.
   g. Save to disk: generate UUID filename `{uuid4()}.png`, write to `MEDIA_DIR/{project_id}/storyboard/{filename}`, using `os.makedirs(storyboard_dir, exist_ok=True)`.
   h. Compute `file_url = f"/media/{project_id}/storyboard/{safe_name}"`.
   i. Check if shot has any existing selected frame: `has_selected = db.query(database.StoryboardFrame).filter(database.StoryboardFrame.shot_id == str(shot_id), database.StoryboardFrame.is_selected == True).count() > 0`.
   j. Create StoryboardFrame record:

      frame = database.StoryboardFrame(
          shot_id=str(shot_id),
