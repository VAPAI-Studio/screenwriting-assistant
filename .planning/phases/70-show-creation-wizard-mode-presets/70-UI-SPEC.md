## Color

Theme tokens come from `:root` HSL CSS variables (Cinematic Amber). Hex equivalents below are derived from the HSL values for reference only â€” implement with the existing Tailwind token classes (`bg-card`, `border-border`, `text-foreground`, `amber-500`, etc.), never raw hex.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 6% 3.5%)` background / `hsl(240 5% 7%)` card | Modal surface (`bg-card`), modal body |
| Secondary (30%) | `hsl(240 4% 14%)` muted / `hsl(240 5% 10%)` input | Unselected preset card hover (`hover:bg-muted/30`), field inputs (`bg-input`), borders (`border-border`) |
| Accent (10%) | `hsl(38 92% 50%)` amber-500 | Selected-state ONLY (see reserved-for list) |
| Destructive | `hsl(0 72% 51%)` | Not used in this phase (no destructive actions) |

Accent reserved for:
- Selected preset card: `border-amber-500/40 bg-amber-500/5 glow-amber` (the `.glow-amber` utility = soft 15px/30px amber box-shadow at 6%/3% opacity).
- Selected preset card icon chip: `bg-amber-500/15 text-amber-400`.
- Selected indicator dot: `w-2 h-2 rounded-full bg-amber-500`.
- Input focus ring (existing): `focus:ring-amber-500/30 focus:border-amber-500/40`.
- Primary CTA button (existing `Button` default variant = `bg-primary`, amber).

Accent is NOT applied to unselected cards, body text, or as a general interactive-element color. Unselected card = `border-border`, neutral muted icon chip (`bg-muted text-muted-foreground`), hover `hover:border-muted-foreground/20 hover:bg-muted/30`.

## Component Inventory

### Preset card section (new â€” in `CreateShowModal`, and reused on edit)

Reuse the `CreateProjectModal` clickable-card pattern verbatim (lines 138â€“163): `<button type="button">` with `flex items-center gap-4 p-4 rounded-xl border`, icon chip (`w-10 h-10 rounded-lg`), title + helper text block, conditional selected dot. Wrap in `space-y-2.5`.

Three cards, in this order. Spanish label (`text-sm font-medium text-foreground`) + English helper line (`text-xs text-muted-foreground mt-0.5`):

| Order | Spanish label (verbatim, D-05) | English helper line | Suggested `lucide-react` icon | Sets `continuity_mode` | Seeds `episode_duration_minutes` |
|-------|--------------------------------|---------------------|-------------------------------|------------------------|----------------------------------|
| 1 | **Microserie** | Short connected episodes that build on each other (~2 min each). | `Zap` | `connected` | **2** |
| 2 | **Serie conectada** | A connected series â€” each episode carries the season arc and prior episodes forward (~22 min each). | `Link` | `connected` | **22** |
| 3 | **AntologĂ­a** | Independent episodes â€” each stands alone, no cross-episode context. | `LayoutGrid` | `anthology` | (none â€” leave duration unset/default) |

Selection model: single-select. Exactly one card selected at a time; selecting a card sets `continuity_mode` and seeds `episode_duration_minutes` per the table. Default on open: no card pre-selected (Create CTA stays disabled until a preset is chosen â€” see interaction states). For the **edit** surface, the card matching the show's current `continuity_mode` is pre-selected on mount (Microserie vs. Serie conectada disambiguated by the stored duration: 2 â†’ Microserie, else connected â†’ Serie conectada).

A short section label above the cards: `Continuity` (`text-xs font-medium text-muted-foreground uppercase tracking-wider mb-2`), matching existing field labels.

### Conditional season-arc reveal (D-07)

When a **connected** preset (Microserie or Serie conectada) is selected, reveal a **Season Arc** field inline, positioned directly below the preset card list and above the modal Actions row. Hidden entirely for **AntologĂ­a** and when no preset is selected.

- Field: `<textarea rows={3}>` reusing the exact input styling from `CreateShowModal` description field (`w-full rounded-lg border border-border bg-input px-3.5 py-2.5 text-sm ... focus:ring-amber-500/30 ... resize-none`).
- Label: `Season Arc` (`text-xs font-medium text-muted-foreground uppercase tracking-wider mb-2`), matching `BIBLE_SECTIONS` "Season Arc" (constants.ts) so it maps cleanly to `bible_season_arc`.
- Placeholder: reuse the bible placeholder â€” `Outline the overarching story arc for the season...`
- Optional, not required â€” empty season arc must not block submit. Reveal/hide animates with the existing `animate-fade-up` (0.5s) keyframe for a non-jarring inline appearance.

### Duration handling (resolves the open "2-minute" question)

**Decision: 2 is seeded as a custom value, NOT added to `DURATION_PRESETS`.** Rationale: adding `2` to the shared `DURATION_PRESETS` is explicitly in Deferred Ideas; the preset only seeds the initial `episode_duration_minutes` value, and the existing `EpisodeDurationPicker` already renders any non-preset value through its "Custom duration" branch (`isPreset === false â†’ showCustom`). The wizard does not need to surface a duration picker inline â€” it sends the seeded value in the `ShowCreate` payload, and the user can later change it via the existing `EpisodeDurationPicker` on the show page (durations stay editable metadata, D-03/D-04).

If the planner chooses to surface duration inline in the modal, it MUST reuse `EpisodeDurationPicker` as-is (no new control) and the 2-min seed will appear as a custom value there â€” no `DURATION_PRESETS` edit.

### Edit-side mode-change control (D-08, Claude's discretion â€” pinned)

**Decision: reuse the identical three preset cards** as the mode-change control on the edit surface (in/adjacent to `BibleEditor`), pre-selected to the show's current mode. Rationale: visual + interaction consistency with creation; one card component, one mental model; no new selector pattern to maintain. Changing the selection issues a `PUT /api/shows/{id}` (React Query mutation) updating `continuity_mode` (and, only if the user has not set a custom duration, the seeded default). Scope is limited to the mode control + season-arc visibility; the full `BibleEditor` section-visibility adaptation remains deferred (D-08).

## Interaction States

| State | Behavior |
|-------|----------|
| Default (modal open) | Title field autofocused. No preset selected. Season-arc field hidden. Create CTA disabled. |
| Hover (unselected card) | `hover:border-muted-foreground/20 hover:bg-muted/30`, 200ms transition. |
| Selected card | `border-amber-500/40 bg-amber-500/5 glow-amber`, amber icon chip, amber dot. Single-select swaps selection. |
| Connected preset selected | Season Arc textarea reveals inline (`animate-fade-up`). |
| AntologĂ­a selected | Season Arc textarea hidden; duration seed cleared/left default. |
| Submitting | Create CTA shows `Creating...` and is disabled (`createShowMutation.isPending`), matching existing modal. |
| Edit: mode changed | `PUT` mutation in flight disables the cards; on success invalidate the show query. |

Validation: submit requires a non-empty `title` (existing) AND a selected preset. Season arc is always optional.
