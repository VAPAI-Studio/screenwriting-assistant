## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-02 | Tampering | client-sent `continuity_mode` in createShow payload | accept | Backend app-validates the `ContinuityMode` enum (Phase 67 D-03) and rejects out-of-enum with 422. The UI only ever sends `connected`/`anthology` (the two wizard presets), well inside the valid set. |
| T-70-03 | Tampering | chained `updateBible` (duration/season arc) | accept | Reuses the existing Phase 37/41 bible update endpoint, already owner-scoped and validated server-side; no new surface introduced. |
| T-70-04 | Denial of Service | two-call create sequence (createShow then updateBible) leaving a show with unseeded duration if the second call fails | mitigate | The show is still created (continuity_mode persisted); a failed `updateBible` only means the duration/season-arc seed did not apply, which the user can set on the show page (durations are editable metadata, D-03/D-04). Surface the create/update failure via the existing inline error copy (UI-SPEC :159) rather than leaving the user without feedback. |
| T-70-SC | Tampering | npm installs | accept | No package installs in this plan (`lucide-react` already a dependency); no legitimacy gate required. |
</threat_model>
