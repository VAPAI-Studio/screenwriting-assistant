    ```python
    class GeneratedShot(BaseModel):
        scene_index: int = Field(description="1-based index of the scene this shot belongs to")
        shot_size: str = Field(default="", description="e.g., Wide, Medium, Close-Up, Extreme Close-Up")
        camera_angle: str = Field(default="", description="e.g., Eye Level, Low Angle, High Angle, Dutch")
        camera_movement: str = Field(default="", description="e.g., Static, Pan, Tilt, Dolly, Tracking, Crane")
        description: str = Field(default="", description="What the camera sees in this shot")
        action: str = Field(default="", description="What happens during this shot")
        script_excerpt: str = Field(default="", description="The specific passage from the script this shot covers")

    class ShotlistGenerationResponse(BaseModel):
        shots: List[GeneratedShot]
    ```

    **2. Internal dataclass:**
    ```python
    @dataclass
    class ShotlistContext:
        screenplay_texts: List[str]       # From ScreenplayContent.content
        scene_summaries: List[dict]       # [{id: str, summary: str, sort_order: int}]
        project_title: str
        existing_user_shots: int          # Count of user_modified shots (for AI context)
    ```

    **3. System prompt constant** `GENERATION_SYSTEM_PROMPT` -- a detailed prompt instructing the AI to act as a professional cinematographer. Must include:
    - Role: professional cinematographer creating a detailed shotlist for production
    - Rules for shot ordering: establishing/wide shots first in each scene, then medium shots, then close-ups; action shots before reaction shots
    - Rules for field population: every shot MUST have non-empty shot_size, camera_angle, camera_movement, description, action, and script_excerpt
    - Rules for scene assignment: use the 1-based scene_index exactly as provided in the scene list
    - Guidance: typically 3-10 shots per scene, more for complex action sequences
    - Constraint: script_excerpt must be a verbatim or near-verbatim passage from the provided screenplay text

    **4. ShotlistGenerationService class** with these methods:

    `_build_context(self, db: Session, project_id: UUID) -> ShotlistContext`:
    - Query ScreenplayContent where project_id matches (same as BreakdownService)
    - Query PhaseData(phase="scenes", subsection_key="scene_list") -> ListItems ordered by sort_order (same as BreakdownService)
    - Query Project for title
    - Count existing shots where user_modified=True for this project
    - Return ShotlistContext dataclass

    `_build_user_prompt(self, ctx: ShotlistContext) -> str`:
    - Format with header: `# Screenplay: {project_title}`
    - Scene list: `## Scenes\nScene 1: {summary}\nScene 2: {summary}\n...` (1-based)
    - Screenplay content: `## Screenplay Content\n{text}\n---\n`
    - Footer: `\nGenerate a complete shotlist covering all scenes.`
    - If ctx.existing_user_shots > 0, add note: `Note: {N} user-edited shots already exist and will be preserved. Generate shots to complement them.`
    - Truncate combined screenplay text at 50000 characters if longer

    `async def _call_ai(self, ctx: ShotlistContext) -> ShotlistGenerationResponse`:
    - Build messages list: [system prompt, user prompt]
    - Call `await chat_completion_structured(messages=messages, response_model=ShotlistGenerationResponse, temperature=0.3, max_tokens=8000)`
    - Return the validated response

    `_map_scene_index_to_id(self, ctx: ShotlistContext, scene_index: int) -> Optional[str]`:
    - Convert 1-based scene_index to 0-based: `zero_based = scene_index - 1`
    - If `0 <= zero_based < len(ctx.scene_summaries)`: return `ctx.scene_summaries[zero_based]["id"]`
    - Else: log warning and return None (same pattern as BreakdownService._map_scene_indices_to_ids)

    `_merge_shots(self, db: Session, project_id: UUID, generated: List[GeneratedShot], ctx: ShotlistContext) -> dict`:
    Smart merge algorithm:
    1. Query all existing shots for this project
    2. Partition: shots with `ai_generated=True AND user_modified=False` -> delete them
    3. Count preserved shots (user_modified=True, or user-created non-AI shots)
    4. For each generated shot:
       - Map scene_index to scene_item_id via _map_scene_index_to_id; skip if None
       - Create new Shot with:
         - `project_id=str(project_id)`
         - `scene_item_id=scene_id`
         - `shot_number` assigned sequentially within each scene (starting from 1)
         - `script_text=gen_shot.script_excerpt`
         - `fields={"shot_size": gen_shot.shot_size, "camera_angle": gen_shot.camera_angle, "camera_movement": gen_shot.camera_movement, "description": gen_shot.description, "action": gen_shot.action}`
         - `sort_order` assigned per scene group (user_modified shots keep position, AI shots fill after)
         - `source="ai"`
         - `ai_generated=True`
         - `user_modified=False`
       - `db.add(new_shot)`
    5. After all inserts, re-number sort_order within each scene:
       - Group all remaining shots by scene_item_id
       - Within each group: user_modified shots keep their relative order first, then AI shots in their generated order
       - Assign sort_order 0, 1, 2, ... sequentially
    6. Return `{"created": N, "deleted": M, "preserved": P}`

    `async def generate(self, db: Session, project_id: UUID) -> dict`:
    Full pipeline:
    1. Build context
    2. Validate: if no screenplay_texts, return `{"status": "error", "message": "No screenplay content found"}`
    3. Validate: if no scene_summaries, return `{"status": "error", "message": "No scenes found. Add scenes before generating shotlist."}`
    4. Call AI
    5. Execute merge
    6. db.commit()
    7. Return `{"status": "success", "shots_created": N, "shots_deleted": M, "shots_preserved": P}`
    Wrap in try/except: on failure, db.rollback(), log error, raise

    **5. Module-level singleton:**
