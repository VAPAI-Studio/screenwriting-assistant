<objective>
Migrate the scene-writing generation path in `_generate_scripts` from JSON-wrapped output (`json_mode=True` returning `{title, content}`) to native screenplay output (`json_mode=False`), so industry-standard formatting (scene headings, action, character cues, parentheticals, dialogue) is no longer degraded by JSON string-encoding. Derive the title from a parseable leading `TITLE:` line, falling back to the scene summary when absent. Strengthen the prompt's layout instructions. Preserve the Phase 45 return contract and continuity/failure behavior exactly.

Purpose: FMT-01 (output preserves formatting without JSON-wrapping degrading it) and FMT-02 (native vs json_mode evaluated, native adopted). Per D-46-04 this is a reasoned adoption with a focused test, NOT a runtime A/B toggle — the native path becomes the single generation shape.

Output: Modified `_generate_scripts` scene call + title parser; updated continuity tests with FMT assertions. No frontend change, no DB/migration, no new dependency.
</objective>
