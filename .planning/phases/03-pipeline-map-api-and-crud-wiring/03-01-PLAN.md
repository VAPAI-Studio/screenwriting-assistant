**1. Expand AgentUpdate schema** in `backend/app/models/schemas.py`:
- Add `system_prompt_template: Optional[str] = Field(None, min_length=50)` to `AgentUpdate` class
- Add `agent_type: Optional[AgentType] = None` to `AgentUpdate` class
- These fields are already present in `AgentCreate`; mirror the same validators/types but make them Optional
- This is required because the Phase 3 success criteria states "Editing an agent's system_prompt_template triggers re-composition" -- impossible without this field in AgentUpdate

**2. Add background helper** at module level in `backend/app/api/endpoints/agents.py`:
- Add imports: `import logging`, `from fastapi import BackgroundTasks`, `from ...db import SessionLocal`, `from ...services.pipeline_composer import pipeline_composer`, `from ...models.database import AgentPipelineMap`
- Add import for schemas: `from ...models.schemas import PipelineMapEntry, PipelineMapResponse`
- Create `logger = logging.getLogger(__name__)` at module level
- Define `async def _recompose_pipeline_background(owner_id_str: str):` that:
  - Creates `db = SessionLocal()`
  - Wraps `await pipeline_composer.compose_pipeline(owner_id_str, db)` in try/except
  - Logs errors via `logger.error("Background pipeline recomposition failed for owner %s: %s", owner_id_str, e)`
  - Always closes `db` in `finally` block
- Follow the `resume_book` pattern from `book_processing_service.py` -- pass string IDs only, create own session

**3. Add GET /pipeline-map endpoint** in `backend/app/api/endpoints/agents.py`:
- Place AFTER the existing `GET /tags` route and BEFORE the `PATCH /{agent_id}` route (critical: avoids FastAPI interpreting "pipeline-map" as an agent_id UUID parameter)
- Route: `@router.get("/pipeline-map")`
- Signature: `async def get_pipeline_map(current_user: schemas.User = Depends(get_current_user), db: Session = Depends(get_db))`
- Query: `db.query(AgentPipelineMap).filter(AgentPipelineMap.owner_id == str(current_user.id)).all()`
- Note: Use `str(current_user.id)` in the filter to match how Phase 2's `compose_pipeline` stores owner_id as a string (cross-DB compatibility decision from Phase 2)
- Convert: `entries = [PipelineMapEntry.model_validate(m) for m in maps]`
- Return: `PipelineMapResponse(owner_id=current_user.id, entries=entries, total_mappings=len(entries))`
