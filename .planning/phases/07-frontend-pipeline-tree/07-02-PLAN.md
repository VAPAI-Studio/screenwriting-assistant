Note: `TreeAgent` MUST include `icon: string` and `agentType: AgentType` fields, matching the `Agent` interface from `types/index.ts`. Import `AgentType` from `'../../types'`.

**Data fetching (3 queries inside the component):**
- `useQuery({ queryKey: [QUERY_KEYS.PIPELINE_MAP], queryFn: () => api.getPipelineMap() })` for pipeline map entries
- `useQuery({ queryKey: [QUERY_KEYS.AGENTS], queryFn: () => api.getAgents() })` for agent metadata (name, color, icon, is_active, agent_type)
- `useQuery({ queryKey: [QUERY_KEYS.TEMPLATE('short_movie')], queryFn: () => api.getTemplate('short_movie') })` for human-readable phase/subsection names

**`buildTreeData` function (defined outside the component, pure function):**
1. Creates an `agentMap: Map<string, Agent>` from the agents array
2. Groups pipeline entries by `phase` then by `subsection_key`
3. Uses `templateConfig.phases` to get the ordering and human-readable names
4. Filters out entries where the agent no longer exists in agentMap
5. Maps agent fields including `icon: agent.icon` and `agentType: agent.agent_type`
6. Returns an array of `TreePhase` objects

Follow the exact implementation from Research Pattern 4 (`buildTreeData` function). The key detail is using `templateConfig.phases` for ordering and filtering, not raw entry iteration.

**Collapsible state:**
- `const [expandedPhases, setExpandedPhases] = useState<Set<string>>(new Set())` -- all collapsed by default
- `togglePhase(phaseId)` toggles a phase in/out of the Set (see Research pattern for the existing codebase convention)

**Toggle mutation:**
```typescript
const toggleMutation = useMutation({
  mutationFn: ({ agentId, isActive }: { agentId: string; isActive: boolean }) =>
    api.updateAgent(agentId, { is_active: isActive }),
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: [QUERY_KEYS.AGENTS] });
    queryClient.invalidateQueries({ queryKey: [QUERY_KEYS.PIPELINE_MAP] });
  },
});
```

Note: The toggle mutation invalidates BOTH `AGENTS` and `PIPELINE_MAP` to satisfy TREE-02 (tree refreshes on agent edits). Even though `is_active` is not a semantic field for recomposition, toggling is an "edit" action that the tree must reflect.

Export the component as `export function AgentPipelineTree()`. For now, render a wrapper div with the "Pipeline Map" header and a placeholder that shows `treeData.length` phases to verify the data layer works. This will be replaced with full rendering in Task 2.

```tsx
export function AgentPipelineTree() {
  // ... queries, buildTreeData, state, mutation as above ...

  const isLoading = pipelineQuery.isLoading || agentsQuery.isLoading || templateQuery.isLoading;
  const treeData = useMemo(() => {
    if (!pipelineQuery.data || !agentsQuery.data || !templateQuery.data) return [];
    return buildTreeData(pipelineQuery.data.entries, agentsQuery.data, templateQuery.data);
  }, [pipelineQuery.data, agentsQuery.data, templateQuery.data]);

  return (
    <div className="mt-6">
      <div className="flex items-center gap-2 mb-3">
        <Network className="h-4 w-4 text-muted-foreground" />
        <h3 className="text-sm font-semibold text-foreground">Pipeline Map</h3>
      </div>
      {/* Scaffold -- Task 2 replaces this with full tree rendering */}
      {isLoading && <div className="animate-pulse bg-muted/40 rounded h-6 w-3/4" />}
      {!isLoading && treeData.length === 0 && (
        <div className="text-center py-6 text-muted-foreground">
          <Network className="h-7 w-7 mx-auto mb-2 opacity-30" />
          <p className="text-xs">Create agents to see how they map to your pipeline</p>
        </div>
      )}
      {treeData.map(phase => (
        <div key={phase.phase} className="text-sm">{phase.phaseName}: {phase.subsections.length} subsection(s)</div>
      ))}
    </div>
  );
}
```

```typescript
function AgentToggleBadge({ agent, onToggle, isToggling }: {
  agent: TreeAgent;
  onToggle: (agentId: string, newState: boolean) => void;
  isToggling: boolean;
}) {
  return (
    <div className={`flex items-center gap-2 px-2.5 py-1.5 rounded-lg border transition-colors ${
      agent.isActive
        ? 'border-border bg-muted/20'
        : 'border-border/50 bg-muted/5 opacity-50'
    }`}>
      <span
        className="w-2.5 h-2.5 rounded-full flex-shrink-0"
        style={{ backgroundColor: agent.color }}
      />
      <span className="text-xs font-medium text-foreground truncate max-w-[80px]">{agent.name}</span>
      <span className="text-xs text-muted-foreground">{Math.round(agent.confidence * 100)}%</span>
      <button
        onClick={() => onToggle(agent.agentId, !agent.isActive)}
        disabled={isToggling}
        className={`ml-auto w-7 h-4 rounded-full transition-colors flex-shrink-0 relative ${
          agent.isActive ? 'bg-emerald-500' : 'bg-muted-foreground/30'
        }`}
      >
        <span className={`absolute top-0.5 w-3 h-3 rounded-full bg-white transition-transform ${
          agent.isActive ? 'left-3.5' : 'left-0.5'
        }`} />
      </button>
    </div>
  );
}
```
