   `ExtractedSceneAppearance(BaseModel)`:
   - `scene_index: int` -- Field(description="1-based index matching the scene list provided in the prompt")
   - `context: str` -- Field(description="Brief description of how this element appears in the scene")

   `ExtractedElement(BaseModel)`:
   - `category: str` -- Field(description="One of: character, location, prop, wardrobe, vehicle")
   - `canonical_name: str` -- Field(description="The standard canonical name for this element, deduplicated across scenes")
   - `description: str` -- Field(description="Brief description of the element as it appears in the screenplay")
   - `scene_appearances: List[ExtractedSceneAppearance]` -- Field(description="Scenes where this element physically appears on screen")

   `ExtractionResponse(BaseModel)`:
   - `elements: List[ExtractedElement]`

2. **ExtractionContext dataclass** (for internal data passing):
   ```python
   @dataclass
   class ExtractionContext:
       screenplay_texts: List[str]        # From ScreenplayContent.content
       character_names: List[str]          # From story.characters ListItem names
       scene_summaries: List[dict]         # [{id: str, summary: str, sort_order: int}]
       project_title: str
   ```

3. **EXTRACTION_SYSTEM_PROMPT** constant: Use the exact prompt from the research document (Pattern: extraction system prompt). This instructs the AI about on-screen-only extraction (EXTR-04), deduplication rules, and the 5 categories.

4. **BreakdownService class** with these methods:

   `_build_extraction_context(self, db: Session, project_id: UUID) -> ExtractionContext`:
   - Query ScreenplayContent records for the project (filter by `project_id == str(project_id)`)
   - Query PhaseData where phase="story", subsection_key="characters", then get ListItems with character names from `content.get("name")`
   - Query PhaseData where phase="scenes", subsection_key="scene_list", then get ListItems ordered by sort_order with summaries from `content.get("summary")`
   - Query Project for title
   - Return ExtractionContext with all gathered data
   - CRITICAL: All UUID comparisons use `str()` cast per project convention

   `_build_user_prompt(self, ctx: ExtractionContext) -> str`:
   - Format: `# Screenplay: {title}`, then `## Known Characters` with bullet list, then `## Scenes` with numbered list (`Scene 1: {summary}`), then `## Screenplay Content` with full text
   - Scene indexing is 1-based (Scene 1, Scene 2, ...) so AI scene_index references map to sort_order

   `async def extract(self, db: Session, project_id: UUID) -> dict`:
   - STUB for now -- just returns `{"status": "not_implemented"}`. Plan 11-02 fills this in.
