<objective>
Make breakdown extraction reliably scene-scoped. Today the AI receives a `## Scenes` 1-based summary list AND a SEPARATE `## Screenplay Content` blob (all scene texts concatenated with `---`, no index labels); it must mentally align the two to self-report each element's `scene_index`. On longer scripts this self-segmentation mis-attributes elements. This plan restructures the user prompt so each scene's FULL text is emitted under its own explicit 1-based header in the same index space `_map_scene_indices_to_ids` and `ExtractedSceneAppearance.scene_index` already use — removing the alignment ambiguity.

Purpose: BFID-02 scene-scoped attribution accuracy, with BFID-01 (full text reaches the AI) verified-already-satisfied and BFID-03 (on-screen-only rules) preserved.

Output: Restructured `_build_extraction_context` + `_build_user_prompt` (plus an alignment helper) in `breakdown_service.py`, and new tests in `test_breakdown_service.py`. SINGLE structured call preserved (D-50-02). No schema change, no migration, no new dependency.
</objective>
