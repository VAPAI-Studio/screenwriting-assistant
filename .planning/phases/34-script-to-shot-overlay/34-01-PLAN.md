    Modify the rendering logic. The existing segments from `buildHighlightSegments` define the text chunks. For each segment, track the cumulative character offset. Within each segment's span, check if any part falls within a `shotRange`. If the ENTIRE segment falls within one shot range, apply the `.shot-overlay` class to that span. If the segment partially overlaps, sub-split it.

    The simplest correct approach: track a running `charOffset` across segments. For each segment:
    - Compute `segStart = charOffset` and `segEnd = charOffset + seg.content.length`
    - Find all shot ranges that overlap `[segStart, segEnd)`
    - If no overlap: render as before (no shot-overlay styling)
    - If the segment is fully covered by one shot range: add `shot-overlay` class to the existing span, and add an `onClick` handler for the popover (only fire if the click target does NOT have `.element-highlight` class, to preserve element-click priority)
    - If partial overlap: sub-split the segment content into covered/uncovered sub-spans. Covered sub-spans get `shot-overlay` class. Uncovered sub-spans render normally. Element-highlight class/props still apply to ALL sub-spans if the segment is type `'highlight'`.

    The onClick handler for shot-overlay spans:
    ```typescript
    const handleShotOverlayClick = (e: React.MouseEvent, shots: Shot[]) => {
      // Don't open popover if user clicked an element highlight
      if ((e.target as HTMLElement).closest('.element-highlight')) return;
      e.stopPropagation();
      const rect = (e.currentTarget as HTMLElement).getBoundingClientRect();
      setPopoverState({ shots, rect });
    };
    ```

    After the segments rendering, conditionally render the popover:
    ```typescript
    {popoverState && (
      <ShotOverlayPopover
        shots={popoverState.shots}
        rect={popoverState.rect}
        onDismiss={() => setPopoverState(null)}
      />
    )}
    ```

    IMPORTANT: The element-highlight `onClick` already calls `e.stopPropagation()`, so when a user clicks on an element-highlight span that also has shot-overlay, the element click fires first and propagation stops — the shot-overlay click on a wrapper span will NOT fire. This is the correct priority (element navigation > shot popover).
