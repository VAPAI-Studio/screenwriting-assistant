# Phase 49 — UI Design Contract

> Visual and interaction contract for the per-scene "Regenerate & Compare" affordance and the `SceneCompareModal`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Hard rule:** This phase MUST reuse the existing "Cinematic Amber" dark theme, the Radix Dialog pattern (CreateProjectModal / AddElementDialog), the `Button` cva variants, and the screenplay paper/courier rendering already in `ScreenplayEditorView.tsx`. No new visual language. No new dependency.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Radix UI + class-variance-authority, no shadcn registry) |
| Preset | not applicable — `components.json` absent; tokens live in `frontend/src/index.css` + `frontend/tailwind.config.js` |
| Component library | Radix UI (`@radix-ui/react-dialog`) |
| Icon library | lucide-react (`Sparkles`/`Wand2` for regenerate, `Loader2` for spin, `X` for close, `Check`/`FileText` reused) |
| Font | `font-screenplay` (Courier Prime → Courier New → monospace) for screenplay text; `font-display` (Playfair Display) for modal title; `font-body` (DM Sans) for UI chrome/labels |

Detected, do NOT redefine: theme is dark-first (default `:root` is the dark Cinematic Amber palette). The `frontend/src/index.css` file also defines alternate palettes (blue, purple) on other selectors — the modal MUST consume the CSS-variable tokens (`bg-card`, `text-foreground`, etc.) so it inherits whichever palette is active, never hard-coded hex except the established paper color `hsl(240,5%,8.5%)`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`/`gap-1.5`), inline pill padding |
| sm | 8px | Compact element spacing, footer button gap (`gap-2.5` ≈ 10px allowed as existing-pattern exception) |
| md | 16px | Pane inner gap, header/footer vertical padding |
| lg | 24px | Modal horizontal padding (`px-6`), header/footer block padding |
| xl | 32px | Column gutter between the two panes on desktop |
| 2xl | 48px | Reserved — not used in this modal |
| 3xl | 64px | Reserved — not used in this modal |

Exceptions (match existing components, do not "fix"):
- Screenplay page inner padding stays `72px 80px` (the `ScreenplayEditorView` paper page constant). The compare panes use a reduced screenplay padding of `40px 48px` because the panes are narrower than a full page — declared exception, applied consistently to BOTH panes so the comparison is fair.
- Footer button gap `gap-2.5` (10px) and toolbar `py-2.5` are inherited from `CreateProjectModal`/`ScreenplayEditorView` and are kept for visual consistency.

## Color

Tokens are HSL CSS variables; the table shows the default dark Cinematic Amber resolution. Always reference the token, not the hex.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 6% 3.5%)` `--background` | Modal scrim base, pane scroll background (`bg-background`) |
| Secondary (30%) | `hsl(240 5% 7%)` `--card` + paper `hsl(240,5%,8.5%)` | Modal surface (`bg-card`), screenplay paper panels, header/footer bands |
| Accent (10%) | `hsl(38 92% 50%)` `--primary` / amber-500 | Reserved list below |
| Destructive | `hsl(0 72% 51%)` `--destructive` | Error-state messaging and retry affordance only |

Accent (amber) reserved for — and ONLY for:
1. The "Keep new version" primary button (`Button variant="default"`).
2. The "Regenerate & Compare" trigger icon + the regenerate/retry action inside the modal.
3. The "NEW" pane badge and the generating-state spinner/progress accent on the right pane.
4. Focus rings (`--ring` is amber by token) and the scene-index pill border (matching the existing `Page x / y` pill style).

Never apply amber to: the "Keep current" button, the left pane chrome, body text, or borders other than focus.

Pane identity (so the eye never confuses old vs new):
- LEFT = current/stored. Header badge: neutral `text-muted-foreground` on `bg-muted/40`, label "CURRENT".
- RIGHT = regenerated. Header badge: amber `text-amber-400` on `bg-amber-500/10` border `border-amber-500/20`, label "NEW".

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Keep new version" |
| Secondary action | "Keep current" |
| Trigger control | "Regenerate & Compare" (per-scene; icon + label, abbreviates to icon-only `aria-label="Regenerate & compare this scene"` when space-constrained) |
| Modal title | "Compare scene {n}: {SCENE TITLE}" (n is `episode_index + 1`) |
| Generating-state heading (right pane) | "Regenerating scene…" |
| Generating-state body | "Rewriting with the improved continuity, voice, and craft passes. This can take up to a minute." |
| Empty/idle (left pane has no stored text — edge) | "No stored text for this scene yet." |
| Error state | "Regeneration failed. Your current version is untouched." + secondary line "{server message, truncated}" |
| Error recovery actions | "Try again" (amber, re-runs regenerate) and "Cancel" (closes modal, no change) |
| Keep-new in progress | "Saving…" (button label while persist mutation pending) |
| Keep-new success → close | No toast required (internal tool); modal closes and the screenplay view re-renders via `SUBSECTION_DATA` invalidation. A brief amber check on the button (`Check` icon, 600ms) before close is permitted. |

Destructive-leaning confirmation: "Keep new version" overwrites the stored scene and marks breakdown + shotlist stale. It is NOT styled destructive (it is the desired primary path), but its tooltip/`title` MUST read: "Replaces the stored scene and flags the breakdown and shotlist as out of date." No extra confirm dialog — the compare itself IS the confirmation surface (the user has read both versions). "Keep current" is a pure no-op close and needs no confirmation.

## Component Spec — Per-Scene Trigger

**Decision: a per-scene-header inline button, NOT a scene picker inside the modal.**

Justification:
- The user is already reading the scene in context (`ScreenplayEditorView` renders scenes as a continuous paginated document with each scene introduced by its uppercased `title`). Targeting "this scene" should be a zero-step action at the point of reading, not a re-selection step.
- A picker inside the modal would duplicate scene-selection logic and break the "judge THIS scene" intent from D-49-04.
- The existing affordance vocabulary already supports inline, low-chrome per-element buttons (the paper page's hover "Click to edit" hint, the toolbar pill/button styles).

Placement & style:
- Anchor the trigger to each scene's title boundary. In view mode, render a small floating control at the top-right of the scene block (mirrors the existing `absolute top-5 right-7` page-number / `top-5 left-7` "Click to edit" hover pattern), revealed/strengthened on `group-hover`, always keyboard-focusable.
- Visual: ghost-style icon+label button matching the toolbar `Edit` button (`text-xs font-medium`, `bg-muted/40 border border-border/40 rounded-lg hover:bg-muted/70`), with an amber `Sparkles`/`Wand2` icon (`h-3.5 w-3.5`) as the accent. Label "Regenerate & Compare" on wide viewports, icon-only with `aria-label` when narrow.
- Clicking it opens `SceneCompareModal` for that scene's `episode_index` and immediately kicks off regeneration (right pane enters generating state on open).
- Must NOT interfere with the existing whole-page "click to edit" affordance: the trigger `stopPropagation` so clicking it does not enter edit mode.

## Component Spec — SceneCompareModal

Built on the exact Radix Dialog skeleton from `CreateProjectModal`/`AddElementDialog`: `Dialog.Root` → `Dialog.Portal` → `Dialog.Overlay` → `Dialog.Content` → `Dialog.Title` + `Dialog.Close`.

**Overlay:** `fixed inset-0 bg-black/60 backdrop-blur-sm data-[state=open]:animate-fade-in z-40` (identical to existing modals).

**Content shell:** `fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2 rounded-xl bg-card border border-border shadow-2xl shadow-black/40 data-[state=open]:animate-scale-in overflow-hidden z-50` — BUT sized large for a two-column read:
- Width: `w-[92vw] max-w-[1100px]`.
- Height: `h-[88vh]` with internal flex column (header / panes / footer); panes scroll, shell does not.

**Header band** (`flex items-center justify-between px-6 py-4 border-b border-border/60 bg-card/95`):
- Left: `Dialog.Title` — `font-display text-xl font-semibold text-foreground`, copy "Compare scene {n}: {TITLE}". Plus a scene-index pill reusing the existing amber pill style (`text-[11px] font-mono text-amber-500/70 bg-amber-500/5 border border-amber-500/10 px-2.5 py-0.5 rounded-full`), reading "Scene {n}".
- Right: `Dialog.Close` icon button (`rounded-lg p-1.5 text-muted-foreground hover:text-foreground hover:bg-muted`, `<X className="h-4 w-4" />`, `aria-label="Close"`). Close is disabled while a keep-new persist is mid-flight; allowed during generating (cancels the regenerate).

**Body — two-column compare** (`flex-1 grid grid-cols-2 gap-8 px-6 py-4 overflow-hidden` on desktop):
- Each column is a flex-column: a pane sub-header (badge "CURRENT" / "NEW" per Color section) + a scrollable screenplay panel below it.
- Each scrollable panel: `flex-1 overflow-auto bg-background rounded-sm` containing the paper panel `bg-[hsl(240,5%,8.5%)] border border-border/30 rounded-sm shadow-2xl` with the declared `40px 48px` padding, rendering text in a `<pre className="font-screenplay text-[13px] text-foreground/90 whitespace-pre-wrap break-words" style={{ lineHeight: '20px' }}>` — exactly the `ScreenplayEditorView` view-mode renderer, minus pagination (panes scroll continuously; pagination is not needed for a single scene).
- Each pane scrolls independently (`overflow-auto` per pane). Min pane width before responsive collapse: 360px.

**Responsive behavior:**
- `< 900px` (`max-md`): collapse from side-by-side to stacked (`grid-cols-1`), CURRENT on top, NEW below, each pane capped at ~40vh with its own scroll, so both remain comparable without full-screen scrolling. The pane badges keep old/new identity. (Internal tool used on desktop primarily; stacked fallback prevents unreadable narrow columns.)

**Footer band** (`flex items-center justify-end gap-2.5 px-6 py-4 border-t border-border/60 bg-card/95`):
- "Keep current" → `Button variant="outline"` (neutral, non-destructive, left of primary).
- "Keep new version" → `Button variant="default"` (amber primary).
- Optional left-aligned status text (`text-xs text-muted-foreground`) for generating/error context.

## States

| State | Trigger | Left pane | Right pane | Footer |
|-------|---------|-----------|------------|--------|
| idle (pre-open) | — | n/a | n/a | Trigger lives in scene header |
| generating | Modal opens → regenerate request + poll runs | Stored text fully rendered, scrollable, fully interactive-read | Centered loading: `Loader2` amber `animate-spin` (or `animate-spin-slow`) + "Regenerating scene…" heading + body copy; optional `shimmer`/`pulse-warm` skeleton lines in screenplay style | "Keep current" enabled; "Keep new version" disabled (`opacity-40`, `disabled`) |
| loaded | Regenerate resolves with new text | Stored text | New text rendered identically to left | Both buttons enabled |
| error | Regenerate request/poll fails | Stored text (preserved, untouched) | Destructive message block: `text-destructive` heading "Regeneration failed. Your current version is untouched." + truncated server message + actions "Try again" (amber) / "Cancel" (ghost/outline) | "Keep current" enabled (acts as safe exit); "Keep new version" disabled |
| keep-current | User clicks "Keep current" or closes | — | — | No mutation. Modal closes. Zero persistence. |
| keep-new (persisting) | User clicks "Keep new version" | dimmed `opacity-60` | dimmed `opacity-60` | Button shows "Saving…", both buttons + Close disabled |
| keep-new (success) | Persist mutation resolves | — | — | Invalidate `QUERY_KEYS.SUBSECTION_DATA(projectId, phase, subsection.key)`; optional 600ms amber `Check`; modal closes; screenplay view re-renders with new scene |
| keep-new (error) | Persist mutation fails | restored to non-dimmed | restored | `text-destructive` "Couldn't save the new version. Your stored scene is unchanged — try again." Buttons re-enabled. The freshly generated text is retained in state so the user can retry keep without re-generating. |

Latency note (informs the generating state, per CONTEXT): a single scene at `max_tokens=4000` can exceed the 30s `API_TIMEOUT`. Use the run/poll pattern or the 120s `CHAT_TIMEOUT`. The generating state must therefore be designed for up to ~60s — hence the explicit "can take up to a minute" body copy and a non-blocking spinner (no fake progress bar).

| Action | Component | Variant | Disabled when |
|--------|-----------|---------|---------------|
| Keep current | `Button` | `outline` | persisting keep-new |
| Keep new version | `Button` | `default` (amber primary) | generating OR error-without-result OR persisting |
| Close (X) | Radix `Dialog.Close` icon button | existing icon-button style | persisting keep-new |
| Try again (error only) | `Button` | `default` (amber) | already retrying |
| Cancel (error only) | `Button` | `ghost` | — |

## Accessibility

- Focus trap + scroll lock + `Esc` to close: provided by Radix `Dialog`. Do not override `onEscapeKeyDown` except to BLOCK escape while a keep-new persist is mid-flight (prevent closing during a write).
- `Dialog.Title` is present and is the accessible name; add `Dialog.Description` (visually `sr-only` if desired) summarizing "Compare the stored scene with a regenerated version and choose which to keep."
- Each pane wrapped in a labelled region: `<section aria-label="Current stored version">` / `<section aria-label="Regenerated version">` so screen-reader users can distinguish panes; the visual CURRENT/NEW badges carry matching text.
- Generating state announces via `aria-live="polite"` ("Regenerating scene…" → "Regeneration complete" / "Regeneration failed").
- All actions are real `<button>`s, reachable by Tab in DOM order: Close → (panes) → Keep current → Keep new version. Footer actions reachable without a mouse.
- Focus ring uses the token `--ring` (amber) via `focus-visible:ring-2 focus-visible:ring-ring/50` from the `Button` base — keep it; do not remove focus styling.
- Trigger button has explicit `aria-label="Regenerate & compare this scene"` for the icon-only/narrow rendering.
- Color is never the sole signal: old/new also carry the text labels "CURRENT"/"NEW"; error also carries text, not just the destructive hue.

## Out of Scope (per CONTEXT D-49 / deferred)

Explicitly NOT in this contract — must not be designed or built here:
- Inline / word-level diff highlighting between the two panes (full-text side-by-side only).
- Version history, N alternates, or a revert timeline (compares exactly two: current vs one regenerated).
- Multi-scene / batch compare (one scene at a time).
- Automated LLM quality scoring of the two versions (user judges by eye).
- Re-threading the global running synopsis on keep-new (per D-49-05 open question, synopsis left untouched).
- Any new route, new dependency, or schema migration.
