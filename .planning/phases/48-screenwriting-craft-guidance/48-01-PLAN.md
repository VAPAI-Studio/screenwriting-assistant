<objective>
Add explicit screenwriting-craft guidance to the per-scene script-writing prompt so generated screenplays
reflect the four named craft dimensions (CRAFT-01): subtext in dialogue, action-line economy, show-don't-tell,
and page pacing / white space. The craft directives are the mechanism by which action lines become visual and
economical (CRAFT-02) and dialogue carries subtext rather than on-the-nose intention statements (CRAFT-03).

Purpose: SC#1/CRAFT-01 require the generation prompt to *explicitly* cover the four craft dimensions by name.
A dedicated, auditable section (per D-48-01) makes that coverage testable and keeps layout-vs-craft concerns
readable. The directives must compose with the Phase 45 continuity block and Phase 47 voice block without
bloating the prompt (SC#4 / D-48-03).

Output: a concise, UNCONDITIONAL `## Screenwriting Craft` block injected additively into the existing
`_generate_scripts` prompt f-string, plus a new test module pinning the anchors and composition.

Scope guardrail: backend-only, single production touch point (`_generate_scripts` prompt). No wizards.py change
(craft needs no new data), no migration, no frontend change, no new dependency. Craft is ADDITIVE — it does not
displace the Phase 45 continuity block, the Phase 47 voice block, or the Phase 46 layout rules (D-48-04).
</objective>
