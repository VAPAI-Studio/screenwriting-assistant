<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env config → app runtime | env vars (MCP_BASE_URL, ALLOWED_ORIGINS) shape server metadata and CORS; supplied by operator/host |
| MCP client → /mcp endpoint | untrusted clients reach the MCP server; auth enforced by static-bearer TokenVerifier |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-01 | Spoofing/Elevation | MCP_BASE_URL in AuthSettings | accept | Base URL is metadata only (issuer/resource_server_url); static-bearer verification lives entirely in ApiKeyTokenVerifier and is unaffected by the URL value. No OAuth server is created. |
| T-62-02 | Information Disclosure | ALLOWED_ORIGINS permissive (localhost) in prod | accept (deferred) | Known deferred risk → Phase 66 (Public-Deploy Hardening) locks CORS to the Vercel domain. This phase keeps the warning-only guard per D-09; not a blocker. |
| T-62-03 | Tampering | misconfigured MCP_BASE_URL via env | accept | Field defaults to localhost; a wrong value affects only advertised metadata, not auth. Operator-controlled env at trusted host boundary. |
</threat_model>
