# v6.0 — Pre-existing Test-Suite Concern (out of scope for phases 45–49)

**Recorded:** 2026-06-06 during autonomous execution of v6.0 (after Phase 47).

## Summary

The backend test suite has a **test-isolation / ordering** problem that predates milestone v6.0. It is NOT a regression introduced by phases 45, 46, or 47. All v6.0 phase tests pass cleanly in isolation.

## Evidence (measured, not inferred)

| Run | Result |
|-----|--------|
| `test_yolo_integration.py` alone | **7 passed** |
| `test_gating_combined` alone | **passed** |
| `test_yolo_integration.py` + `test_session_isolation.py` together | **1 failed, 11 passed** |
| `test_continuity_generation.py` + `test_character_voice_injection.py` + `test_wizard_injection.py` | **21 passed** |
| Full suite, certain orderings | up to **7 failed** (all in test_yolo_integration + test_session_isolation) |
| **Baseline `aa623f3`** (pre–Phase 45), yolo+session_isolation | **1 failed, 11 passed** — identical to current |

## Root cause (characterized, not yet fixed)

- `test_yolo_integration.py` tests pass in isolation but fail when run after other modules → **module-level / singleton state leakage or DB-session bleed** between test modules (the `sqlalchemy` "extra_forbidden" / pydantic collection errors are pollution artifacts, not logic failures).
- The one genuinely order-independent failure — `test_session_isolation.py::test_orchestrate_uses_session_factory` — **already failed at baseline `aa623f3`** (before any v6.0 work). Pre-existing.

## Disposition

- **Not blocking v6.0 phases.** Phases 45–47 each pass their own suites with zero regressions; verified by reverting production files to baseline and reproducing the identical pre-existing failures.
- **Clarifies prior reports:** Phase 47's executor reported "4 pre-existing failures" and the verifier observed "7" — both were measuring order-dependent pollution, not distinct code defects. The true order-independent failure count attributable to nothing in v6.0 is **1** (`test_orchestrate_uses_session_factory`, pre-existing at baseline).

## Recommended follow-up (separate maintenance task, not a v6.0 phase)

- Add proper per-test teardown / fixture isolation for the AI-provider and DB-session singletons so `test_yolo_integration.py` and `test_session_isolation.py` are order-independent.
- Triage `test_orchestrate_uses_session_factory` (failing since before v6.0).
- Consider `pytest-randomly` in CI to surface ordering coupling deterministically.
